# 🌴 Developers-Paradise

**A Full-Stack AI-Powered Pain Point & Idea Discovery Platform**

Developers-Paradise is designed to autonomously hunt down developer complaints, software limitations, and genuine product ideas across the internet, analyze them using state-of-the-art LLMs, and present them in a highly-interactive Next.js dashboard. It bridges the gap between developers experiencing friction and builders looking for validated SaaS or open-source ideas.

---

## 🏗 System Architecture

The project consists of two core engines seamlessly working together:

1. **The Ingestion Pipeline (Python, AI, GitHub Actions)**
   An autonomous data scraper that sweeps the web for developer pain points.
2. **The Web Dashboard (Next.js 16, Prisma, Postgres)**
   The user-facing platform where founders and developers can discover, vote on, discuss, and track the ideas generated by the pipeline or submitted manually.

- **Email/Newsletters**: Resend

## 📡 The Autonomous Ingestion Pipeline

Found in the `ingestion/` directory, this pipeline acts as the engine of Developers-Paradise. 

### How it Works
1. **Multi-Source Scraping**: Powered by `scrapers.py`, the system fetches live data from:
   - **GitHub**: GraphQL API searching for open enhancement issues with high reactions.
   - **StackExchange**: Polling top voted questions in popular tags (python, reactjs, etc.).
   - **Discourse Forums**: Bypassing pagination to extract feature requests natively.
   - **Hacker News**: Using the Firebase API to read "Ask HN" threads and their top comments.
   - **Reddit**: Searching specific tech subreddits (r/SaaS, r/webdev, etc.) for "pain point" queries.
   - **Lobste.rs**: Fetching hottest tech threads.
   - **Next.js Hydration**: Grabbing structured DOM data from specific SSR-rendered blogs.
2. **Deduplication**: `pipeline.py` compares retrieved URLs against the existing Postgres database to ensure we don't process the same issue twice.
3. **AI Standardization**: Gemini 2.5 Pro analyzes the raw text to determine if it truly represents a software pain point. If valid, Gemini transforms the unstructured rant into a highly structured JSON object (Title, Description, Difficulty, Dev Time, Domain, Stack, Tags).
4. **Embedding Generation**: Gemini's Embedding 001 model converts the idea into a 3072-dimensional vector.
5. **Database Storage**: The structured JSON and the vector embedding are inserted into the Postgres database.

### Automation
The ingestion pipeline runs completely autonomously via a **GitHub Action** (`.github/workflows/ingestion.yml`). It is scheduled to run every day at Midnight UTC, ensuring a continuous fresh feed of ideas. It also triggers a weekly Friday newsletter route.

## 💻 The Web Platform

Found in the `src/` directory, this is the Next.js frontend where users interact with the data.

### Core Features
- **Idea Dashboard**: View dynamic feeds of AI-curated and user-submitted ideas.
- **Trending & Leaderboards**: See which pain points are resonating most with the community by tracking Upvotes and Waitlist sign-ups.
- **Deep Dives**: Individual `idea/[id]` pages featuring difficulty tags, estimated dev times, recommended tech stacks, and source links.
- **Community Interaction**: 
  - Save ideas to your profile (`SavedIdea`).
  - Join waitlists for ideas you'd use (`Waitlist`).
  - Comment and discuss solutions (`Comment`).
  - Declare you are building it (`Repository`) and toggle `openToCoFounders`.
- **User Profiles**: Manage your saved items, tracked repositories, and community standings.
- **Admin Controls**: Manage platform health and manually trigger actions.

## ⚙️ Local Setup Guide

To run Developers-Paradise locally, you need Node.js, Python 3.11+, and a PostgreSQL server (with pgvector installed).

### 1. Clone the Repository
```bash
git clone https://github.com/Teraxyl14/Developers-Paradise.git
cd Developers-Paradise
```

### 2. Environment Variables
Create a `.env` file in the root directory. You will need:
```env
# Database
DATABASE_URL="postgresql://user:password@localhost:5432/problemsite"

# NextAuth Secrets
AUTH_SECRET="your-nextauth-secret-key"

# AI & Scraping (Required for Python Pipeline)
GEMINI_API_KEY="your-gemini-api-key"
GITHUB_GRAPHQL_TOKEN="your-github-pat"

# Emails
RESEND_API_KEY="your-resend-api-key"
```

### 3. Database Setup (Prisma)
Ensure your Postgres database is running and has the `vector` extension enabled.
```bash
npm install
npx prisma generate
npx prisma db push
```
*(Optionally run the seed script if you want initial mock data via `src/app/api/seed/route.ts` if implemented)*

### 4. Running the Web Platform
Start the Next.js development server:
```bash
npm run dev
```
Access the dashboard at [http://localhost:3000](http://localhost:3000).

### 5. Running the Ingestion Pipeline (Standalone)
If you want to manually test the AI web-scraper locally:
```bash
# Setup Python Virtual Environment (Recommended)
python -m venv venv
source venv/bin/activate  # Or `venv\Scripts\activate` on Windows

# Install Dependencies
pip install -r ingestion/requirements.txt

# Run the Pipeline
python ingestion/pipeline.py
```

## 🤝 Contributing
Want to add a new scraper source (e.g., Discord, X/Twitter)? 
Open `ingestion/scrapers.py`, inherit the `Scraper` base class, build your extractor, and append it to `get_all_scraped_data()`. Everything else (AI parsing, DB insertion) is handled automatically!
